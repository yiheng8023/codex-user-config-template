# Contributing

Thank you for helping improve this template.

## Public-safe rule

Do not contribute real personal configuration, memory snapshots, credentials, account state, local machine paths, browser/session data, logs, caches, or private preferences.

## Good contributions

- Safer template structure.
- Clearer setup documentation.
- Better validation checks.
- Public-safe examples and placeholders.
- Cross-platform improvements.

## Pull request checklist

- [ ] The change contains no secrets or personal data.
- [ ] Examples are placeholders, not real user state.
- [ ] `python -B scripts/verify.py` passes.
- [ ] License and third-party provenance are clear.
